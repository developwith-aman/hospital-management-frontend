# Hospital Management System – Frontend

## Overview

This is the frontend module of the Hospital Management System developed using Java Swing.  
The frontend provides a graphical user interface (GUI) through which users can interact with the backend services for managing patients, doctors, appointments, departments, billing, and authentication.

The frontend communicates with the Spring Boot backend using REST APIs over HTTP.

# Frontend Architecture

The frontend follows a layered structure:

## Panels
Responsible for GUI rendering and user interaction.

## Service Layer
Handles communication with backend APIs.

## DTO Classes
Transfer data between frontend and backend.

## Utility Classes
Provide reusable API handling and JWT token management.

---

# API Communication

The frontend communicates with the backend using REST APIs.

Example workflow:

1. User clicks a button in the GUI
2. ActionListener triggers the request
3. Service layer prepares the DTO
4. API client sends HTTP request
5. Backend processes the request
6. Response is displayed in the GUI

---

# JWT Authentication

After successful login:

- JWT token is generated by the backend
- Token is stored in the frontend session
- Every protected request includes the token in the Authorization header

Example:

Authorization: Bearer <JWT_TOKEN>
